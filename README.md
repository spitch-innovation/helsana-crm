# 🏥 Helsana CRM Search API & Conversational Tool

A robust, AI-powered API and frontend integration tool built for the Helsana CRM Hackathon.

This project solves the problem of locating customer records in a CRM using messy, incomplete, or phonetically misspelled data (e.g., from Automatic Speech Recognition transcripts). It leverages `gpt-4o-mini` to perform **Fuzzy Matching** directly against a lightweight, in-memory JSON corpus extracted from the core CRM data.

---

## 🏗 Architecture Overview

The system consists of three main components:

1. **The FastAPI Backend (`helsana-crm.py`)**: A Python servlet that loads the CRM CSV on startup, cleans the data, and exposes endpoints for searching and retrieving verification questions.
2. **The LLM Matcher**: Instead of strict database queries or complex Levenshtein algorithms, the backend passes raw search strings (or broken JSON) to OpenAI, asking it to identify the correct `PARTNERNR` based on context.
3. **The Conversational Tool (Frontend JS)**: A tool definition and execution function used by an AI voice/chat agent. It collects identifying information from the user and queries the backend.

## 🚀 Quickstart: Setting up the API

### 1. Prerequisites

Ensure you have Python 3.9+ installed and an active OpenAI API key.

### 2. Installation

Navigate to the project directory and install the required dependencies:

```bash
pip install fastapi uvicorn pandas openai pydantic

```

### 3. Environment Variables

Export your OpenAI API key so the backend can authenticate with the LLM matcher.

**Mac/Linux:**

```bash
export OPENAI_API_KEY="sk-your-openai-key-here"

```

**Windows (Command Prompt):**

```cmd
set OPENAI_API_KEY="sk-your-openai-key-here"

```

### 4. Running the Server

Start the FastAPI server using `uvicorn`.

```bash
python3 helsana-crm.py

```

*The server will start on `http://0.0.0.0:8003`.*

> **📖 API Documentation:** > Once running, you can access the interactive Swagger UI documentation automatically generated by FastAPI by visiting: `http://localhost:8003/docs`

## 🔍 How the Search Works

The core `/search` endpoint is designed to be highly resilient to bad data.

### 1. Data Prep & In-Memory Corpus

On startup, the FastAPI app loads `data/helsana-crm.csv`. It cleans the headers, fills missing values, and extracts only the `SEARCHABLE_COLUMNS` into a JSON string (`SEARCH_CORPUS_JSON`). This keeps the data lightweight and ready for the LLM context window.

### 2. The Fuzzy Search (`/search`)

The endpoint accepts *any* raw text body. This allows the frontend to send strict JSON, or just a messy ASR string like `"last name is Müller, email is at my mail dot com"`.

The LLM is prompted to perform robust fuzzy matching against the `SEARCH_CORPUS_JSON` and return the matching `PARTNERNR`(s).

### 3. Outcomes & Verification

The API processes the LLM's matches and groups them by household (`FAMILY_ID`). It returns an `outcome`:

* `none`: No matches found.
* `multi`: Multiple distinct people matched (requires the agent to ask for more info).
* `unique`: Exactly one person matched.

**If the outcome is `unique**`, the API automatically generates up to 2 context-aware **Verification Questions** (e.g., "What is your selected franchise?") based strictly on the data available in that user's rows.

## 🛠 Integrating the Conversational Tool

To use this API within an AI Voice/Chat agent, provide the agent with the Tool Definition and the Execution Function.

### Tool Definition (Prompt & Schema)

Provide this prompt to your conversational agent so it knows *when* and *how* to use the search tool:

```text
Use this tool to identify a Helsana caller in the CRM using whatever identity details the caller provides.  

This tool is for customer lookup only. Collect any available identifying information from the caller, such as customer ID, first name, last name, mobile number, email address, date of birth, or address details, and send those values to the CRM search service.

Use this tool when:
- the caller introduces themselves,
- the caller provides personal details,
- you need to locate the correct CRM record,
- you need to narrow down none / multiple / unique customer matches.

Important behavior:
- Send whatever identifying fields you have; you do not need every field.
- Use it again if the first search returns no match or multiple matches and the caller provides more details.
- The tool returns the CRM search result, including outcome, hits, verification questions, and post-verification summary data.
- Do not treat the caller as verified just because the tool returns a unique result.
- Verification is handled later in the conversation using the returned verification questions.
- Do not reveal sensitive account details before verification succeeds.
- NEVER EVER provide the answers to the security questions to the customer. NEVER. EVER

```

**JSON Schema:**

```json
{
  "type": "object",
  "properties": {
    "address": {
      "type": "object",
      "properties": {
        "plz": { "type": "string", "description": "Postal code." },
        "strasse": { "type": "string", "description": "Street name." },
        "hausnummer": { "type": "string", "description": "House number." }
      },
      "description": "Caller's address details, if provided.",
      "additionalProperties": false
    },
    "birthday": { "type": "string", "description": "Caller's date of birth." },
    "last_name": { "type": "string", "description": "Caller's last name." },
    "first_name": { "type": "string", "description": "Caller's first name." },
    "customer_id": { "type": "string", "description": "Caller's customer ID if provided." },
    "email_portal": { "type": "string", "description": "Caller's email address." },
    "tel_portal_mobil": { "type": "string", "description": "Caller's mobile phone number." }
  },
  "additionalProperties": false
}

```

### Execution Function (Backend Retrieval)

Use this Javascript function to execute the API call when the agent triggers the tool. It cleans the arguments and handles the network request to the FastAPI server.

```javascript
async (args) => {
  const FASTAPI_BASE_URL = "http://127.0.0.1:8003";

  const cleanString = (value) => {
    if (value === null || value === undefined) return "";
    return String(value).trim();
  };

  const cleanObject = (obj) => {
    const out = {};
    for (const [key, value] of Object.entries(obj || {})) {
      if (value === null || value === undefined) continue;

      if (typeof value === "string") {
        const trimmed = value.trim();
        if (trimmed !== "") out[key] = trimmed;
        continue;
      }

      if (typeof value === "object" && !Array.isArray(value)) {
        const nested = cleanObject(value);
        if (Object.keys(nested).length > 0) {
          out[key] = nested;
        }
        continue;
      }

      out[key] = value;
    }
    return out;
  };

  try {
    const payload = cleanObject({
      customer_id: cleanString(args.customer_id),
      first_name: cleanString(args.first_name),
      last_name: cleanString(args.last_name),
      tel_portal_mobil: cleanString(args.tel_portal_mobil),
      email_portal: cleanString(args.email_portal),
      birthday: cleanString(args.birthday),
      address: args.address
        ? {
            strasse: cleanString(args.address.strasse),
            hausnummer: cleanString(args.address.hausnummer),
            plz: cleanString(args.address.plz)
          }
        : undefined
    });

    if (Object.keys(payload).length === 0) {
      return JSON.stringify({
        success: false,
        error: "At least one customer identity field must be provided."
      });
    }

    const response = await fetch(`${FASTAPI_BASE_URL}/search`, {
      method: "POST",
      headers: {
        "Content-Type": "application/json"
      },
      body: JSON.stringify(payload)
    });

    const text = await response.text();

    if (!response.ok) {
      return JSON.stringify({
        success: false,
        error: `CRM lookup failed with status ${response.status}`,
        details: text
      });
    }

    let result;
    try {
      result = JSON.parse(text);
    } catch (parseError) {
      return JSON.stringify({
        success: false,
        error: "CRM lookup returned invalid JSON.",
        details: text
      });
    }

    return JSON.stringify({
      success: true,
      outcome: result.outcome || "none",
      hits: Array.isArray(result.hits) ? result.hits : [],
      verification_questions: Array.isArray(result.verification_questions)
        ? result.verification_questions
        : [],
      post_verification_summary: result.post_verification_summary || null
    });
  } catch (error) {
    return JSON.stringify({
      success: false,
      error: error?.message || String(error)
    });
  }
}

```

## 🧪 Testing with cURL

You can easily test the fuzzy matching capabilities directly from your terminal. Notice how the endpoint handles the missing capitalization gracefully:

```bash
curl -X POST "http://localhost:8003/search" \
    -H "Content-Type: application/json" \
    -d '{"last_name": "tarka"}' \
    | python3 -m json.tool

```

Expect a rich JSON response detailing the outcome, the matched customer record, dynamically generated verification questions, and context payloads for downstream intent handling.
